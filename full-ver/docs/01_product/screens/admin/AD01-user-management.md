# AD01. ユーザー/ロール管理

**画面ID**: AD01
**画面名**: ユーザー/ロール管理
**対象ロール**: ADMIN のみ
**機能**: ユーザーの CRUD と権限管理

---

## 目的

管理者がユーザーを作成・編集・無効化し、ロール・Layer・ユニットを管理する。

---

## 画面仕様

### ナビゲーション

- **パンくずリスト**: H01 > AD01

### セクション 1: ユーザー一覧

**表示列**

| 列 | 説明 |
|----|------|
| 名前 | display_name |
| メール | email |
| ロール | PLANNER / AGENT / EXEC / ADMIN |
| Layer | Member / Manager（AGENT/PLANNER のみ） |
| ユニット | 所属ユニット名 |
| ステータス | active / inactive（バッジ） |
| アクション | 編集, 無効化, 削除 |

**ソート**:
- 名前（あいうえお順）
- 作成日（新順）
- ステータス（有効優先）

**フィルタ（オプション）**:
- ロール: チェックボックス
- ステータス: active / inactive
- ユニット: マルチセレクト

### セクション 2: ユーザー作成/編集フォーム（モーダル）

**入力項目**

| 項目 | 型 | 必須 | 説明 |
|------|------|------|------|
| 名前 | text | ✓ | display_name |
| メール | email | ✓ | ログイン用メール |
| パスワード | password | ✓（新規）| bcrypt でハッシュ化 |
| ロール | select | ✓ | PLANNER / AGENT / EXEC / ADMIN |
| Layer | select | - | Member / Manager（AGENT/PLANNER のみ表示） |
| ユニット | select | ✓（AGENT/PLANNER） | 1 ユーザー → 1 ユニット |
| ステータス | select | ✓ | active / inactive |

---

## CRUD 操作

### Create（新規作成）

1. 「+ 新規ユーザー」ボタン → モーダルオープン
2. 上記フォーム入力
3. 「作成」をクリック
4. User レコード作成
5. メール送信: 初期パスワード + ログインリンク（オプション）

### Read（一覧表示）

- 全ユーザーを表示（フィルタ・ソート可）

### Update（編集）

1. 一覧から「編集」ボタン → モーダルオープン
2. 各フィールドを編集
3. 「保存」をクリック
4. User レコード更新
5. 監査ログに記録

### Delete（無効化）

1. 一覧から「無効化」ボタン → 確認ダイアログ
2. 確認後、status = inactive に変更
3. ログイン不可になる

**注**: 物理削除はしない（監査ログのため）

## Given/When/Then

### GWT-AD01-001: Admin が新規ユーザーを作成

```gherkin
Given Admin が AD01 を開く
When 「+ 新規ユーザー」をクリック
And モーダルで「name: 田中太郎」「email: tanaka@example.com」「role: AGENT」「layer: Member」「unit: 営業チーム」を入力して「作成」をクリック
Then User レコードが作成される
And 一覧に「田中太郎」が表示される
And ステータスは「active」
```

### GWT-AD01-002: Admin がユーザーのロールを変更

```gherkin
Given ユーザー「田中太郎」（role: AGENT）が存在
When 編集モーダルでロールを「EXEC」に変更して「保存」をクリック
Then User.role が「EXEC」に更新される
And 監査ログに記録される
```

### GWT-AD01-003: Admin がユーザーを無効化

```gherkin
Given ユーザー「田中太郎」（status: active）が存在
When 一覧から「無効化」をクリック → 確認ダイアログで「確認」をクリック
Then User.status = inactive に変更される
And ログイン不可になる
```
