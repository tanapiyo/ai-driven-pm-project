# AD04. 監査ログ

**画面ID**: AD04
**画面名**: 監査ログ
**対象ロール**: ADMIN のみ
**ルート**: `/admin/audit-logs`
**Phase 2 追跡**: SCR-104 として計画された機能が MVP に実装済み

---

## 目的

管理者がシステム内のすべての重要な操作履歴（作成・更新・削除等）を確認する。

---

## 画面仕様

### レイアウト

- **ヘッダー**: "監査ログ"
- **説明文**: "システム内のすべての重要な操作履歴を確認できます"
- **フィルターセクション**: AuditLogFilter コンポーネント
  - 期間フィルター
  - 操作種別フィルター（オプション）
- **ログテーブル**: AuditLogTable コンポーネント

| 列       | 説明                 |
| -------- | -------------------- |
| 日時     | 操作が行われた日時   |
| ユーザー | 操作を行ったユーザー |
| 操作     | 実施された操作の種別 |
| 対象     | 操作対象のリソース   |
| 詳細     | 変更内容の概要       |

- **ページネーション**: Pagination コンポーネント（URL パラメータ同期）

### URL パラメータ

| パラメータ | 説明                 | デフォルト       |
| ---------- | -------------------- | ---------------- |
| `page`     | ページ番号           | 1                |
| `limit`    | 1 ページあたりの件数 | デフォルト設定値 |
