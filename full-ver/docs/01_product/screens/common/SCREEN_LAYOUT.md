# 共通画面構成（全画面統一）

**全画面で遵守する統一的なレイアウト・ナビゲーション・UX パターン**

---

## 概要

すべての画面は **2 カラムレイアウト** を採用：
- **左サイドバー** (240px): グローバルナビゲーション
- **メインエリア**: 機能コンテンツ + ブレッドクラム

---

## 1. グローバルナビゲーション（左サイドバー）

### 構成要素

#### A. ロゴ・ホームボタン（最上部）

```
┌────────────────────┐
│  🏠 App Name          │ ← クリックで H01（ホーム）へ
└────────────────────┘
```

- **アクション**: クリックで H01 に遷移
- **表示**: 全ロール共通

#### B. 権限別メニュー（スクロール可能）

**メニュー項目は RBAC ロールに基づいて表示制御**:

| メニュー | 対象ロール | 説明 | 遷移先 | 順序 |
|---------|----------|------|--------|------|
| **インフルエンサー検索** | PLANNER, AGENT, EXEC, ADMIN | 検索・発見フロー | S01 | 最上位（ダッシュボード直下） |
| **担当管理** | AGENT | 自担当インフルエンサー管理 | E01 | 2番目（Agent） |
| **ユニット管理** | AGENT/Manager, EXEC, ADMIN | ユニット管理 | U01 | 3番目（Agent）, view-all（Exec/Admin） |
| **商流管理** | AGENT, EXEC, ADMIN | 商流管理 | 未実装 | 4番目（Agent）, view-all（Exec/Admin） |
| **カバレッジ分析** | EXEC | 経営ダッシュボード | M01 | 2番目（Exec main） |
| **関係値分析** | AGENT, EXEC | 関係値スコア管理 | Phase 2 | - |
| **ユーザー管理** | ADMIN | ユーザー CRUD | AD01 | - |
| **マスタ管理** | ADMIN | Channel, Genre, Rank, Segment | AD02 | - |
| **管理者ダッシュボード** | ADMIN | 管理者機能 | /admin | 最下位（Admin） |

**表示ルール**:
- ロールに該当するメニューのみ表示
- 例: PLANNER → インフルエンサー検索 のみ
- 例: AGENT → 担当管理, 関係値分析, ユニット管理

**アクティブ状態**:
- 現在ページのメニューは **ハイライト（背景色 + アイコン強調）**
- 例: S01 を開いている時、「インフルエンサー検索」がハイライト

#### C. ボトムメニュー（固定）

```
┌────────────────────┐
│ 🌓 テーマ切替       │ ← ライト/ダーク/システム
├────────────────────┤
│ ⚙️ 設定             │ ← /settings へ遷移
├────────────────────┤
│ 👤 ユーザー情報     │ ← アバター、名前、ロール表示
├────────────────────┤
│ ログアウト          │ ← セッション終了
└────────────────────┘
```

**ユーザーメニュー**:

| メニュー | 説明 | 遷移先 |
|---------|------|--------|
| **設定** | アカウント情報、外観設定 | /settings |
| **ログアウト** | セッション終了 | - |

**設定ページ内のセクション**:

| セクション | 説明 | パス |
|-----------|------|------|
| **アカウント** | ユーザーID、ロール表示、パスワード変更 | /settings/account |
| **外観** | テーマ設定（ライト/ダーク/システム） | /settings/appearance |

## 2. メインエリア（コンテンツ）

### A. ヘッダー

#### ブレッドクラム

```
ホーム > インフルエンサー検索 > インフルエンサー一覧
```

**ルール**:
- 最初のレベル: 常に「ホーム」
- 階層に応じて パン粉を追加
- 各パン粉はクリック可能（遡り遷移）
- 現在ページは **太字 + アクティブ色**

#### グローバルアクション（右上）

```
[ユーザー名 ▼] [通知 🔔] [ヘルプ ❓] [設定 ⚙️]
```

---

### B. コンテンツ領域

- **マージン**: 左右 24px, 上下 24px
- **最大幅**: 1400px（超大画面対応）
- スクロール: 独立スクロール（サイドバーは固定）

## 3. 画面遷移と戻る機能

### パターン A: サイドメニューからの遷移（リセット）

```
S01 画面 → メニュー「担当管理」クリック → E01 画面
```

- **挙動**: 履歴はリセット、新しい画面へジャンプ
- **戻る**: ブラウザの戻るボタン OR サイドメニューで前のメニューを選択

### パターン B: ドリルダウン遷移（履歴保持）

```
S01（検索ホーム）→ S02（一覧）→ S03（詳細）
```

- **挙動**: 各画面にブレッドクラムと「戻る」ボタン
- **戻る**:
  - ブレッドクラムをクリック（複数段階遡り可）
  - 「← 戻る」ボタン（1 段階戻る）
  - ブラウザ戻るボタン

### パターン C: モーダル遷移

```
E01（担当管理）→ A03 モーダル（実績登録）
```

- **挙動**: 背景はグレーアウト、モーダルは最前面
- **戻る**:
  - 「キャンセル」ボタン
  - モーダル外をクリック（ESC キー対応）
  - モーダルが閉じて E01 に戻る

#### オーバーレイ仕様

| 属性 | 値 |
|------|-----|
| 背景色（ライト） | `bg-black/50`（50% 透過の黒） |
| 背景色（ダーク） | `dark:bg-black/60`（60% 透過の黒） |
| 目的 | 元ページが見えることで作業のコンテキストを維持 |
| z-index | `z-50` |
| 閉じる操作 | ESCキー、オーバーレイクリック、キャンセルボタン |

## 4. レスポンシブ対応

### デスクトップ（≥ 1024px）

```
┌─────────────────────────────────┐
│ サイド（240px） │ メイン（~1400px）  │
└─────────────────────────────────┘
```

- サイドバー: 固定、常に表示
- メイン: スクロール

### タブレット（768px ～ 1023px）

```
┌──────────────────────────┐
│ サイド（200px）│ メイン   │
└──────────────────────────┘
```

- サイドバー: 幅を圧縮（アイコンのみ表示）
- メニューテキストはツールチップ表示

### モバイル（< 768px）

```
┌──────────────────┐
│ ハンバーガー ☰   │ ← クリックで スライドイン
├──────────────────┤
│ メイン           │
└──────────────────┘
```

- サイドバー: ハンバーガーメニュー（スライドイン）
- ハンバーガーアイコンはヘッダー左上に固定
- メイン: フルスクリーン

## 6. 権限による表示制御

### ログインユーザーのロール確認

各画面ロード時に、ログインユーザーの `role` および `layer` を確認：

```typescript
// サイドバーメニュー表示ロジック
const visibleMenus = menus.filter(menu => {
  switch(menu.requiredRole) {
    case 'PLANNER':
      return user.role === 'PLANNER' || user.role === 'ADMIN';
    case 'AGENT':
      return user.role === 'AGENT' || user.role === 'ADMIN';
    case 'EXEC':
      return user.role === 'EXEC' || user.role === 'ADMIN';
    case 'ADMIN':
      return user.role === 'ADMIN';
    default:
      return true;
  }
});
```

**ADMIN の扱い**:
- ADMIN は全メニュー表示
- ロールスイッチ機能（管理上、他ロール視点でシステム動作確認）は Phase 2

## 7. Given/When/Then（共通画面構成）

### GWT-LAYOUT-001: ロール別メニュー表示

```gherkin
Given Planner がログイン済み
When H01（ホーム）を開く
Then サイドバーに以下が表示される:
  - インフルエンサー検索
And 以下は非表示:
  - 担当管理（Agent のみ）
  - ユーザー管理（Admin のみ）
```

### GWT-LAYOUT-002: メニューアクティブ状態

```gherkin
Given Planner が S01 を開く
When サイドバーメニューを確認
Then 「インフルエンサー検索」がハイライト表示（背景色 + 強調）
And 他のメニューはデフォルト表示
```

### GWT-LAYOUT-003: ブレッドクラムで遡り遷移

```gherkin
Given S03（インフルエンサー詳細）を開いている
When ブレッドクラムの「インフルエンサー一覧」をクリック
Then S02 に遷移して、検索条件が保持されている
And ブレッドクラムが更新される（S03 が削除される）
```

### GWT-LAYOUT-004: 設定ページにアクセス

```gherkin
Given どの画面でもログイン状態
When サイドバー下部の「⚙️ 設定」をクリック
Then /settings ページに遷移する
And 左側に設定ナビゲーション（アカウント、外観）が表示される
And 右側にコンテンツエリアが表示される
```

### GWT-LAYOUT-005: モーダルでの戻り動作

```gherkin
Given E01（担当管理）を開いている
When 「実績登録」ボタンをクリック → A03 モーダルが表示
And モーダルの「キャンセル」をクリック
Then モーダルが閉じ、E01 に戻る
And E01 のコンテンツは変更されていない
```
