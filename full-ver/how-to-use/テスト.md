# テスト担当とフロー（E2E 含む）

## 1. テスト種別と担当

| テスト種別 | 誰が書く | いつ | 誰が実行 |
|------------|----------|------|----------|
| ユニットテスト | implementer | 実装中（TDD: RED→GREEN→REFACTOR） | test-runner |
| E2E テスト（Playwright） | implementer | タスクに E2E が含まれる場合の実装中 | e2e-runner |
| テスト計画・設計 | qa-planner | Spec/AC 確定後、実装前（設計のみ） | — |

---

## 2. エージェントの役割

| エージェント | 役割 |
|--------------|------|
| **implementer** | ユニットテスト・E2E テストを書く。Workflow の Step 4「Write/update tests」。TDD で RED→GREEN→REFACTOR を回す。 |
| **test-runner** | テストを実行するだけ（lint → typecheck → test → build）。テストコードは書かない。 |
| **qa-planner** | テスト計画・設計のみ。AC → テストケースのマッピング、`docs/03_quality/test-plan/<id>.md` 作成。READ-ONLY。 |
| **e2e-runner** | E2E を実行。`git diff` で影響範囲を推定し、`./tools/contract e2e` で実行、失敗分析まで実施。E2E テストコードは書かない。 |

---

## 3. 人間の役割

- **AC（受け入れ基準）の定義**: 人間が Issue に記述
- **AC の検証**: PR 作成後の人間レビューで、AC が満たされているか確認

---

## 4. フロー上のタイミング

1. Spec / Plan / AC 定義（人間 or AI）
2. テスト計画作成: qa-planner（本プロジェクトのみ）※設計のみ、コードは書かない
3. 実装 + テスト: implementer が TDD でテストを書く
4. 品質ゲート: test-runner が `./tools/contract test` を実行
5. E2E 実行: e2e-runner が `./tools/contract e2e` を実行（本プロジェクトのみ）
6. AC レビュー: 人間が PR をレビューして AC 達成を確認

---

## 5. 本プロジェクト vs テンプレート

### 本プロジェクト（codama-ldh）

| 項目 | 状態 |
|------|------|
| エージェント | 10種（implementer, test-runner, qa-planner, e2e-runner, code-reviewer, architect, designer 等） |
| テスト計画 | qa-planner が設計 |
| E2E | e2e-runner が実行・分析 |
| ワークフロー | DocDD, TDD, /autopilot, /epic-autopilot, /parallel-implement |

### テンプレート（piyo-work/template）

| 項目 | 状態 |
|------|------|
| エージェント | 3種のみ（implementer, test-runner, code-reviewer） |
| qa-planner | なし |
| e2e-runner | なし |
| ワークフロー | SDD、implementer がユニットテストを書く |

テンプレートでは qa-planner と e2e-runner がなく、ユニットテストは implementer が書き、test-runner が実行する構成。

---

## 6. 関連コマンド

| コマンド | 説明 |
|----------|------|
| `./tools/contract test` | ユニットテスト（全パッケージ） |
| `./tools/contract test watch` | TDD 用 watch モード |
| `./tools/contract test coverage` | カバレッジ付きテスト |
| `./tools/contract e2e` | Playwright E2E（`./tools/contract up` 必須） |
| `./tools/contract e2e --scope smoke` | 最小限の smoke（デフォルト） |
| `./tools/contract e2e --scope affected` | 変更影響範囲のみ |
| `./tools/contract e2e --scope full` | 全 E2E スイート |
