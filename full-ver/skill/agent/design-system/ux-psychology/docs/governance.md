# UX Psychology Pack Governance

責任分界・更新頻度・運用ルールを定義する。

---

## Ownership

| 責任範囲 | 担当 | 説明 |
|----------|------|------|
| パック全体のメンテナンス | デザインリード / PdM | テンプレート・プロセスの改善判断 |
| ソース更新（クロール） | 開発チーム（ローテーション） | 月次の更新PR作成 |
| テンプレートレビュー | デザインリード + 開発チーム | 四半期の有効性評価 |
| ダークパターン監査 | PdM + デザインリード | リリース前の最終確認 |

---

## Update Schedule

### Monthly: ソース更新

- **頻度**: 月1回（月初）
- **担当**: ローテーション
- **手順**:
  1. `python scripts/crawl_shokasonjuku_ux_psychology.py --check-robots` で robots.txt を確認
  2. `python scripts/crawl_shokasonjuku_ux_psychology.py` でクロール実行
  3. `python scripts/build_catalog.py` でカタログ再生成
  4. `python ci/check_pack_integrity.py` で整合性チェック
  5. 差分を確認し、PR を作成
- **PR テンプレート**:
  ```
  [Maintenance] UX Psychology Pack 月次ソース更新 (YYYY-MM)

  - 新規概念: X件
  - 更新概念: Y件
  - 削除概念: Z件
  - 整合性チェック: PASSED
  ```

### Quarterly: テンプレートレビュー

- **頻度**: 四半期1回
- **担当**: デザインリード + 開発チーム
- **確認項目**:
  - [ ] テンプレートが実際の開発で活用されているか
  - [ ] 不要なチェック項目がないか
  - [ ] 不足しているチェック項目がないか
  - [ ] ダークパターン防止チェックが最新か
  - [ ] プロセスの改善提案があるか

---

## Escalation

| 状況 | アクション |
|------|-----------|
| ソースサイトの構造変更でクローラーが動作しない | クローラーを修正するIssueを作成 |
| ライセンス・利用規約の問題が判明 | 即座にソースを削除し、PdMに報告 |
| ダークパターンの指摘を受けた | リリースを止め、Impact Assessment を再評価 |
| クロール先がダウンしている | 次月に延期（既存ソースは有効） |

---

## Metrics

パックの有効性を測定するための指標：

| 指標 | 目標 | 測定方法 |
|------|------|----------|
| Impact Assessment 作成率 | UI変更PRの 50%+ | PR テンプレートのチェック状況 |
| ダークパターン検出数 | レビューで0件 | PR レビューコメント |
| テンプレート利用率 | 月次で増加傾向 | テンプレートリンクの PR 数 |

---

## Change Management

パックへの変更は以下のプロセスに従う：

1. **Minor（テンプレート文言修正等）**: PR レビュー1名で可
2. **Major（プロセス変更、新概念カテゴリ追加等）**: デザインリード + PdM 承認
3. **Breaking（ディレクトリ構造変更等）**: チーム全体の合意
