# UX Psychology Acceptance Criteria Template

UX心理学の概念を適用する機能の受入基準テンプレート。
通常の AC に加えて、以下の UX 心理学固有の AC を追加する。

---

## UX Psychology AC Pattern

### AC-UX-001: 概念の適用が意図通りに機能する

- **Given**: ユーザーが [対象画面/機能] を使用している
- **When**: [概念が適用された UI 要素] が表示される
- **Then**: [期待されるユーザー行動/反応] が確認できる

### AC-UX-002: ダークパターンが含まれていない

- **Given**: [概念] が適用された画面が表示されている
- **When**: ユーザーが操作を拒否/中断しようとする
- **Then**: 明確で容易な方法で拒否/中断できる

### AC-UX-003: アクセシビリティが確保されている

- **Given**: 支援技術（スクリーンリーダー等）を使用している
- **When**: [概念が適用された UI 要素] に到達する
- **Then**: 同等の情報と操作が提供されている

### AC-UX-004: 効果測定の基盤が整っている

- **Given**: 機能がリリースされている
- **When**: 定義された測定期間が経過する
- **Then**: [測定指標] のデータが収集可能な状態である

---

## Usage

1. 上記パターンから該当する AC を選択
2. `[...]` のプレースホルダーを具体的な内容に置き換え
3. Issue の AC セクションに追加
4. 必要に応じてカスタム AC を追加

## Example

```markdown
### AC-UX-001: 社会的証明による信頼性向上

- Given: ユーザーがプロダクト詳細ページを閲覧している
- When: レビュー件数とレーティングが表示される
- Then: ユーザーが他のユーザーの評価を確認でき、購買判断の参考にできる

### AC-UX-002: 社会的証明にダークパターンがない

- Given: レビューセクションが表示されている
- When: ユーザーがレビューの詳細を確認する
- Then: すべてのレビューが実際のユーザーからのものであり、虚偽のレビューが含まれていない
```
