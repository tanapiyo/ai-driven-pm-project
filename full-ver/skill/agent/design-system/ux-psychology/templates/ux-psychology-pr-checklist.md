# UX Psychology PR Checklist

PR レビュー時に UX心理学の観点から確認するチェックリスト。

---

## Impact Assessment

- [ ] UX Impact Assessment が作成されている（または「該当なし」の理由が記載されている）
- [ ] 適用した概念が `rules/catalog.json` のエントリに紐付いている
- [ ] 適用しなかった概念と理由が記録されている

## Dark Pattern Prevention

- [ ] ユーザーの目的達成を支援する設計になっている
- [ ] ユーザーが十分な情報のもと自由に選択できる
- [ ] 心理的圧力（偽の希少性、時間制限等）を不当に使用していない
- [ ] 解約・キャンセル・拒否の操作が明確でアクセスしやすい
- [ ] オプトイン/オプトアウトの選択がフェアである

## Accessibility

- [ ] 色だけに頼った情報伝達をしていない
- [ ] フォーカス順序が論理的である
- [ ] スクリーンリーダーで操作可能である
- [ ] WCAG AA コントラスト比（4.5:1 以上）を満たしている

## Measurement

- [ ] 適用した概念の効果を測定する方法が定義されている
- [ ] 測定の目標値が設定されている（定量の場合）
