# Skill: 週次進捗報告書作成（メイン）

## 概要
これは週次進捗報告書を作成するメインのオーケストレータースキルです。個別のセクション生成スキルを順次実行し、最終的に完全な週次進捗報告書を統合して生成します。

## 使用方法

### オプション1: 完全自動実行
このスキルを実行すると、以下のサブスキルを順次実行します：

1. `collect-git-data` - Git履歴データ収集
2. `generate-progress` - 実績レポート生成（セクション2）
3. `analyze-schedule` - スケジュール分析（セクション4）
4. `analyze-risks` - 課題・リスク分析（セクション6-7）
5. `plan-next-week` - 次週計画生成（セクション10）
6. 最終統合 - 完全な週次進捗報告書を生成

### オプション2: 個別実行
各サブスキルを個別に実行することも可能です：

```
# 個別実行の例
/weekly-report:collect-git-data    # Git履歴収集のみ
/weekly-report:generate-progress   # 実績レポート生成のみ
/weekly-report:analyze-schedule    # スケジュール分析のみ
/weekly-report:analyze-risks       # 課題・リスク分析のみ
/weekly-report:plan-next-week      # 次週計画生成のみ
```

## 実行フロー

### ステップ1: データ収集（自動）

```bash
# outputディレクトリ準備
mkdir -p 05_進捗報告/output

# Git履歴収集スクリプト実行
cd 05_進捗報告
./scripts/collect_weekly_data.sh > output/weekly_data.txt
```

### ステップ2: セクション別生成（順次実行）

各セクションを順番に生成します：

#### 2-1. 実績レポート（セクション2）
- **入力**: Git履歴データ（`output/weekly_data.txt`）
- **プロンプト**: `プロンプト/Git履歴から進捗レポート自動生成.md`
- **出力**: `output/section2_実績.md`
- **内容**: 開発活動サマリー、実装完了機能、修正・改善項目、作業中の項目

#### 2-2. スケジュール分析（セクション4）
- **入力**: WBS、残タスク情報、過去の進捗トレンド
- **プロンプト**: `プロンプト/遅延リスク判定.md`
- **出力**: `output/section4_スケジュール.md`
- **内容**: リスクレベル判定、定量分析、推奨アクション、シミュレーション

#### 2-3. 課題・リスク分析（セクション6-7）
- **入力**: 課題管理表、リスク管理表
- **プロンプト**: `プロンプト/課題リスク分析.md`
- **出力**: `output/section6-7_課題リスク.md`
- **内容**: 課題の優先度付け、リスクの定量評価、対応策提案、エスカレーション判定

#### 2-4. 次週計画（セクション10）
- **入力**: WBS、今週の進捗、メンバー稼働状況
- **プロンプト**: `プロンプト/次週計画生成.md`
- **出力**: `output/section10_次週計画.md`
- **内容**: 重点タスク選定、予定会議、リスク対策、クリティカルパス確認

### ステップ3: 最終統合

生成された各セクションと、ユーザーが手動で入力する情報を統合して、完全な週次進捗報告書を作成します。

#### 自動生成セクション
- セクション2: 今週の実績（`section2_実績.md`）
- セクション4: スケジュール状況（`section4_スケジュール.md`）
- セクション6-7: 課題・リスク管理（`section6-7_課題リスク.md`）
- セクション10: 次週の予定（`section10_次週計画.md`）

#### 手動入力が必要なセクション
ユーザーに以下の情報を質問します：

- **セクション1**: エグゼクティブサマリー
  - 総合ステータス（🟢/🟡/🔴）
  - 今週のハイライト（2-3項目）

- **セクション3**: BPR関連進捗（該当プロジェクトのみ）
  - 業務分析・フロー作成状況
  - 業務改善効果の見込み
  - ペインポイント解消状況

- **セクション5**: 予算状況
  - 予算消化状況
  - 予算超過リスク

- **セクション8**: 品質状況
  - レビュー実施状況
  - 不具合状況

- **セクション9**: 変更管理
  - 変更要求一覧
  - 承認済み変更

- **セクション11**: KPI（主要成果指標）

- **セクション12**: その他特記事項

- **セクション13**: 添付資料

#### 最終出力

完全な週次進捗報告書を以下に保存：
```
05_進捗報告/output/週次進捗報告書_YYYY-WXX.md
```

## 使用上の注意

1. **初回実行時**
   - スクリプトに実行権限を付与: `chmod +x 05_進捗報告/scripts/*.sh`
   - outputディレクトリが自動作成されます

2. **データの正確性**
   - Git履歴は自動取得されますが、WBS・課題管理表は最新の状態を準備してください
   - 数値データは生成後に必ず確認してください

3. **カスタマイズ**
   - プロジェクト固有の用語は事前にプロンプトファイルに追加可能
   - クライアントの好みに応じて表現を調整できます

4. **週次ルーチン**
   - 推奨実行タイミング: 毎週金曜日 14:00-15:00
   - 月曜午前のクライアントMTGで報告

## トラブルシューティング

### Q: Git履歴が取得できない
```bash
# リモートから最新を取得
git fetch origin
git pull origin main

# データ収集を再実行
cd 05_進捗報告
./scripts/collect_weekly_data.sh > output/weekly_data.txt
```

### Q: セクション生成に失敗した
各セクションは独立しているため、失敗したセクションのみ再実行可能です。

### Q: 手動入力データが多すぎる
セクション3（BPR）、5（予算）、8（品質）、9（変更管理）は、前週のデータをコピーして差分のみ更新することで効率化できます。
