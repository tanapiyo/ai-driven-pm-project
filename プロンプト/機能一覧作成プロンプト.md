```
# 機能の抽出方法

## 1. 業務ステップから機能を抽出

各業務ステップについて：
- 「システムで自動化する」と記載されているステップ → 機能として抽出
- 「システムで管理する」と記載されているステップ → 機能として抽出
- 手動作業は対象外（システム化しない）

## 2. 機能の分類

抽出した機能を以下の大分類に分類：
- **マスタ管理**: マスタデータの登録・更新・削除・照会
- **データ入力**: 業務データの新規登録・編集
- **承認**: 承認フロー、ワークフロー
- **検索・照会**: データの検索・表示
- **集計・分析**: データの集計・レポート出力
- **外部連携**: 他システムとのデータ連携
- **バッチ処理**: 自動実行される処理
- **セキュリティ**: 認証・認可・監査
- **システム管理**: ユーザー管理、設定管理

## 3. 優先度の設定

各機能の優先度を以下の基準で設定：
- **必須**: システム稼働に必須、業務が止まる
- **重要**: 業務効率に大きく影響、早期実装が望ましい
- **任意**: あると便利だが、後回しでも問題ない

## フェーズ別実装計画（提案）

### 第1フェーズ（MVP: Minimum Viable Product）
- 機能ID: F-001, F-002, F-003（必須機能のみ）
- 目的：最低限の業務が回るようにする

### 第2フェーズ
- 機能ID: F-010, F-011（重要機能）
- 目的：業務効率を向上させる

### 第3フェーズ
- 機能ID: F-020, F-021（任意機能）
- 目的：利便性を向上させる

# 注意事項
- To-Be業務フローに明記されている機能のみを抽出
- 「あると良い」という推測は避ける（クライアントと確認が必要）
- 機能は具体的に記述（「データ管理」ではなく「顧客データ新規登録」）
- 機能IDは連番で採番
```

---

## 使い方

1. To-Be業務フローまたはTo-Be業務記述書を用意
2. プロンプトに貼り付けてAIに実行
3. 出力された機能一覧をレビュー
4. クライアントと一緒に優先度を確認・調整
5. フェーズ分けを決定
